## Website Report

### 1. Structure of a Webpage

A webpage is built using **HTML** (HyperText Markup Language). Every HTML document begins with a `<!DOCTYPE html>` declaration, which tells the browser what type of document it is reading.

The document is then divided into two main sections inside the `<html>` tag:

- **The `<head>` section:** This is not visible on the page itself. It contains information *about* the page, such as the `<title>` (which appears in the browser tab), links to stylesheets, and meta information for search engines.
- **The `<body>` section:** This is where all the visible content goes, headings, paragraphs, images, lists, links, and everything the user can see and interact with in the browser window.

HTML uses *tags* which come in pairs, an opening tag such as `<p>` and a closing tag such as `</p>`. The content sits between them. Some tags, like `<br/>` (line break) and `<hr/>` (horizontal rule), are self-closing and do not need a closing tag.

Colours on a webpage use the **RGB (Red, Green, Blue)** colour system. Each colour is made by mixing different amounts of red, green, and blue light. In HTML, colours can be written using *RGB values*, such as `rgb(255, 255, 255)` for white or `rgb(255, 0, 0)` for red. Opacity can also be added using RGBA, for example `rgba(46, 109, 164, 0.6)`.

### 2. Features of a Good Website

A good website should have the following features:

- **Clear layout and design:** The page should be easy to read, with text and images well organised. The background colour should not clash with the text colour.
- **Easy navigation:** Visitors should be able to find their way around the site easily. Navigation links should be visible and consistent across all pages.
- **Working links:** All internal and external links should work correctly. Dead links are a sign of poor quality and frustrate users.
- **Appropriate use of images:** Images should be the right size and include ALT text for accessibility. Too many large images slow a website down.
- **Consistent style:** The same fonts, colours, and layout style should be used across all pages so the site feels professional and unified.
- **Fast loading:** Pages should load quickly. Heavy graphics or too many animations can make a website very slow.
- **Readable text:** Font size, font style, and colour should make reading easy. Using all CAPS or mixing too many styles makes text hard to read.

### 3. Paper-Based vs Computer-Based Systems

Data can be stored and retrieved using either a paper-based system or a computer-based system. Each has its advantages and disadvantages:

| Feature | Paper-Based System | Computer-Based System |
| --- | --- | --- |
| Storage space | Takes up physical space (folders, filing cabinets) | Can store thousands of files digitally |
| Speed of retrieval | Slow - must search through physical files | Fast - use search function to find instantly |
| Risk of loss | Can be lost, damaged, or destroyed | Can be backed up and protected |
| Cost | Low initial cost (paper, pens, folders) | Higher initial cost (hardware, software) |
| Editing data | Must rewrite or use correction fluid | Easy to edit and update at any time |

> *Overall, computer-based systems are more efficient for storing and retrieving large amounts of data, while paper-based systems are simpler and require no technology or power supply.*
